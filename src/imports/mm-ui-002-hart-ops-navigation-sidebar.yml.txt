context: |
  The sidebar is Hart Ops' strategic command center — each navigation item maps to a
  strategic question: "How are my tenants doing?" (Organizations), "What's happening now?"
  (Events). The Dashboard is the default landing page, designed to deliver situational
  awareness in seconds. As a Hart Ops administrator, I need a persistent navigation
  structure so I can quickly switch between platform monitoring concerns.

scope:
  in_scope:
    - "Fixed left sidebar structure and behavior"
    - "Navigation items and their mapping to platform sections"
    - "Dashboard landing page with stat cards and growth chart"
  out_of_scope:
    - "Detail pages for Organizations (see MM-UI-003)"
    - "Cross-org event monitoring detail (see MM-UI-004)"
    - "Reports and data quality detail (see MM-UI-005)"
    - "Responsive/mobile sidebar behavior (not specified in source)"

key_concepts:
  - term: "Stat Card"
    definition: "A compact visual tile on the Dashboard that displays a single key metric with its label and value. Five stat cards provide at-a-glance platform health."
    sources:
      - doc: "Hart Ops Super Admin Spec"
        details: "§2 Navigation & Sidebar — Dashboard"
  - term: "Platform Growth Chart"
    definition: "A chart on the Dashboard visualizing platform growth trends over time. Specific chart type and data granularity not detailed in source."
    sources:
      - doc: "Hart Ops Super Admin Spec"
        details: "§2 Navigation & Sidebar — Dashboard"

core_content:
  - section_title: "Sidebar Structure"
    points:
      - "Elements: Fixed left sidebar that persists across all pages."
      - "Elements: Each navigation item rendered as collapsed icon + label."
      - "Elements: Main content area occupies the remaining horizontal space to the right of the sidebar."
      - "Design principle: The sidebar is always visible — no hamburger menu or collapsible state specified for desktop."
    sources:
      - doc: "Hart Ops Super Admin Spec"
        details: "§2 Navigation & Sidebar — Shared Layout"

  - section_title: "Navigation Items"
    points:
      - "Functionality: Dashboard — default landing page after sign-in."
      - "Functionality: Organizations — links to the client management interface (list view)."
      - "Functionality: Events — links to the Cross-Org Monitoring view ('omni view')."
      - "Functionality: Reports — links to platform-wide stats with PDF/Excel export capability."
      - "Functionality: Settings — links to team management and profile screens."
      - "Design principle: Each nav item maps to a strategic question Hart needs answered — navigation is organized by concern, not by data entity."
    sources:
      - doc: "Hart Ops Super Admin Spec"
        details: "§2 Navigation & Sidebar"

  - section_title: "Dashboard Landing Page"
    points:
      - "Elements: Five stat cards — Total Orgs, Active Events, Top Orgs, Recent Events, Platform Growth chart."
      - "Functionality: Dashboard loads as the default page after successful authentication."
      - "Functionality: Stat cards display real-time or near-real-time platform metrics."
      - "Design principle: Situational awareness in seconds — a Hart Ops user should understand platform state immediately upon login without clicking into any section."
    sources:
      - doc: "Hart Ops Super Admin Spec"
        details: "§2 Navigation & Sidebar — Dashboard"

assumptions:
  - "The sidebar is fixed-width and does not collapse or expand on desktop viewports."
  - "Active navigation state is visually indicated (highlight, background change, or similar) though the specific style is not defined in source."
  - "Dashboard stat cards refresh on page load; real-time polling or WebSocket updates are not specified."
  - "Platform Growth chart is a time-series visualization — exact chart type (line, area, bar) to be determined during design."
  - "'Top Orgs' and 'Recent Events' cards likely contain short ranked lists rather than single numeric values."

implications:
  - "Frontend: Sidebar must be a persistent layout shell component that wraps all authenticated routes."
  - "Frontend: Dashboard requires five distinct stat card components — some display single values, others display ranked lists or charts."
  - "Frontend: The Platform Growth chart requires a charting library or component."
  - "Backend: Dashboard API must aggregate metrics across all organizations efficiently — this is a cross-tenant query."
  - "Backend: 'Top Orgs' ranking criteria needs definition (by event count? by data quality? by activity?)."
