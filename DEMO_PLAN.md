## How to Frame the Demo

Christopher was explicit about how to approach this presentation. **Walk through user flows, not a feature showcase.**

> **Source — Call 3/4 [00:26:28]:** Christopher recommended "focusing on walking through a user flow instead of just showcasing capabilities" for the stakeholder sessions.

> **Source — Call 4/4 [00:41:56]:** Christopher said: "if we show this they're going to ask okay what's next what's next what's next so I want to get ahead of that — at least show this is where we're at, kind of get some initial feedback [...] in just knowing that where we're checking off the boxes"

## Recommended Demo Flows (Priority Order)

### Flow 1: Event Creation + Campaign Creation + Usage of Activity Templates

**Why show this:** This is the highest-impact feature for stakeholders who deal with massive event volumes. It directly addresses the #1 pain point — managing tens to hundreds of events under campaigns instead of one-by-one manual input.

**What to show:**

- Create an event on the Staff platform — walk through the full creation workflow
- **During event creation, show the Intelligent Report Preview** that appears when selecting objectives — how the system can project expected outcomes (brand awareness score, sales drive, education driver). Frame as the foundation for prescriptive analytics
- Show the event detail page with all collected information and status
- Create a campaign (selecting supplier, channel, geography) — show how events roll up under campaigns
- Show campaign detail page with centralized view of all events (drafts, active, completed)
- Create an Activity Template within the campaign — demonstrate how selecting the template pre-fills the Create Event workflow for high-volume operations

**Rationale from meetings:**

> **Source — Call 3/4 [00:19:41]:** Christopher praised the Activity Template feature: "perfect for replicating campaigns where activities are largely the same, requiring only minor one-off changes during launch."

> **Source — Call 3/4 [00:27:25]:** "A primary concern for the stakeholders will be managing a massive volume of events (tens to hundreds), not just one-offs."

> **Source — Call 3/4 [00:34:52]:** Christopher confirmed that the maximum number of events an operator might input at once is "in the hundreds, and this is currently done manually one by one."

> **Source — Call 1/4 [00:38:42]:** Christopher: "I love this. It's going to be very interesting to flip the script on the way everyone thinks because the business currently is just around executions. It's not about being prescriptive of the best executions and where and timing."

> **Source — Call 1/4 [00:40:02]:** "If we could be analytically persuasive and show the value [...] this is perfect."

> **Source — Call 1/4 [00:42:01]:** On AI integration: "Now the AI can learn more about the types of accounts [...] and be more prescriptive: 'we're doing this campaign and we think you should do these accounts on these days.'"

**Validation question for stakeholders:** Does the campaign hierarchy (Campaign > Activities > Events) match how you think about organizing your work today? What metrics matter most to you when evaluating campaign/event success?

### Flow 2: Educator Assignment (Distance + Availability)

**Why show this:** Christopher emphasized this is the most time-consuming step in the current process and "getting it right will benefit everything else." This flow directly impacts the educator managers (close to the stakeholders being invited).

**What to show:**

- Educator Manager dashboard
- Event with "unassigned" status
- Clicking to assign → seeing list of educators sorted by distance to the event location
- Showing educator availability calendar
- Confirming assignment → notification sent to educator

**Rationale from meetings:**

> **Source — Call 3/4 [00:39:21]:** Christopher said "refining this flow is critical because it currently consumes a large amount of time, and getting it right will benefit everything else."

> **Source — Call 4/4 [00:34:48]:** Christopher on distance: "We take their home address to know the drive distance or travel distance to an event because [...] we don't want to assign them to drive across New York City in rush hour."

> **Source — Call 1/4 [00:24:09 area]:** Christopher stressed the importance of educators constantly updating their availability: "one of the things that we're instituting is from a process standpoint to have the educators communicate via the app more."

**Validation question for stakeholders:** Is proximity-based sorting the right default for educator assignment? Are there cases where you'd override this (e.g., skill-based, brand certification)?

### Flow 3: Event Finalization Queue (Manager Review)

**Why show this:** Christopher specifically asked for the ability to edit educator notes and questionnaire responses before approving. This is a quality control step that stakeholders will recognize as critical.

**What to show:**

- Finalization queue with all completed events pending review
- Opening an event to review data, photos, educator notes
- Ability to edit free-form text before approving (language cleanup)
- Approving and finalizing the event

**Rationale from meetings:**

> **Source — Call 4/4 [00:31:09]:** Christopher: "The free form text — we definitely need opportunity for the manager to be able to change that before approving and submitting so that there isn't any [inappropriate language]. Things like 'the customers hate it' or 'they said it was yucky' [...] just taking out and changing some of the notes."

> **Source — Call 4/4 [00:30:00]:** Christopher asked specifically: "on this approve and finalize, are they able to go in there and change the educator notes and the questionnaire information?"

**Validation question for stakeholders:** What level of editing should managers have? Only free-text, or also the hard-coded questionnaire responses?

## What NOT to Deep-Dive During the Demo

Christopher gave explicit guidance on what to avoid:

1. **Complex Reports** — "Don't dive too deep into complex reports. For the MVP, mimic the reports in the current application and focus on core capabilities." (Call 3/4 [00:13:50])

2. **Survey AI Tool** — Christopher has his own AI survey tool launching separately. "Don't spend too much time spit polishing this because I've got an outline already." (Call 1/4 [00:15:08])

3. **Billing & Budgets** — Important future capability but "advised against building it out immediately, prioritizing the current process flow first." (Call 3/4 [00:41:14])

4. **Data Hierarchy / Schema Details** — Stakeholders are "visually inclined" and care about flows, not data architecture. (Call 4/4 [00:09:52])

5. **SAS vs Affiliate vs Core distinctions** — These are structural/technical. Stakeholders want to see their daily workflow improved. (Call 4/4 [00:16:00])

## Key Validation Points for Stakeholders

Beyond the per-flow questions above, these are the big-picture validations:

1. **Current capabilities coverage:** Are we replicating everything that matters from Hems? Is anything critical missing?
2. **Process flow accuracy:** Does the flow (Email Request > Draft > Campaign > Event > Assignment > Execution > Finalization) match reality?
3. **Role boundaries:** Does the Operator / Admin / Educator Manager / Educator split make sense for your team sizes?
4. **Volume handling:** With 100+ events at a time, does the campaign + template approach feel like a genuine improvement?
5. **Priority for Phase 2:** After the MVP, what should come first — affiliate onboarding, deeper reporting, billing/budgets, or AI integrations?

## Summary

The demo should tell one story: **"We're not just rebuilding Hems. We're building a platform that handles your real daily workflow — from the moment an event request arrives in your inbox to the moment the final report is locked."**

Lead with the pain points they live every day (manual input, phone-based coordination, paper forms), show how each flow solves them, and close with the vision (prescriptive analytics, AI-driven intelligence).
